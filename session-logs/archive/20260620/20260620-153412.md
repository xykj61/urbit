## Thinking trace

User asked whether Zig lacks hyphen handling and if renaming `index-of` → `indexof` was a mistake.

Clarified: Zig allows hyphens in identifiers via `@""` (e.g. `@"index-of"`). The failure was Rishi's expression parser: `-` is subtraction in `evalAddSub`. Prefix `index-of text needle` hits `findLastArithAdd` and parses as `index - of …`. Infix `text index-of needle` works like `starts-with` when `findWordOp` runs before arithmetic — but comparisons with `==` need parentheses because `findComparison` routes the left arm through `evalAddSub`.

Restored infix `index-of`; fixed needle offset (`idx + 10` for `" index-of "`). Tests use parens for `==` comparisons. Renaming to `indexof` was a pragmatic workaround, not a vocabulary goal — hyphenated infix builtins are the Rishi pattern.
