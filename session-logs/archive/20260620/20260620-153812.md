## Thinking trace

User asked whether the expression parser can handle hyphens correctly (not a Zig limitation). Two bugs:

1. **Comparison order** — infix word ops (`index-of`, etc.) ran before `==`, so needles swallowed the right side (`"world" == 6`). Fix: `findComparison` runs first; each arm recurses via `evalExpr` (not `evalAddSub`).

2. **Arithmetic `-` vs word hyphen** — `findLastArithAdd` treated `-` in `index-of` as subtraction. Fix: `isWordHyphen` — skip `-` when both neighbors are ASCII letters (letters only, so `10-3` stays subtraction).

Added `parser_hyphens.rish`; relaxed paren requirement in `index_of.rish`. All string/arithmetic regression tests green; parity 24/24.
