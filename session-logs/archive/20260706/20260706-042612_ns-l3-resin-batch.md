## The Invariants, Stated Before the Frame

The resin-batch frame carries cargo between peers, so its invariants come first. One signature covers the header and the manifest, and it is verified once per batch; payloads carry no signature of their own, because each payload proves itself against the digest that already rides inside the signed manifest. A bad batch signature, a tampered payload, an unknown Tilak, and a duplicated name each refuse the whole crossing, and the refusal lands before a single leaf is appended. Every check passes before any append begins, so a batch either lands whole or lands not at all. Have-already entries carry no bytes; the receiver reconstructs them from resins it already holds, keeping the bytes that stayed home exactly as authoritative as the bytes that crossed. Everything stays bounded by a fixed entry count and a fixed buffer.

## What Was Built

`mantra/resin_batch.rye` grows the frame the resins counsel gave whole as spec I1, now built for the sync. `buildBatch` walks a source bolt revision and, given the digests the receiver already holds, writes a signed header and manifest as the control plane and a payload section as the data plane — held entries marked as refs with no bytes, the rest carrying their bytes once. `applyBatch` reverses it in the disciplined order: parse the manifest, reject an unknown Tilak or a duplicated name, verify the one signature over header and manifest, verify each payload against its digest, and only then append every leaf, reconstructing the held ones from local resins. The read stays `recall`, the unit stays `bolt`, the content stays `resin`, and the vessel stays the amphora — no new name was minted; the two hardcoded Tilak roots the counsel named carry the plain cases.

The witness sends three resins under one signature. One is had-already and crosses as a ref; two are need-resin and cross once, nine payload bytes in all. After apply, the receiver recalls all three identically to the source, including the reconstructed have-already leaf, so referential transparency holds across the frame. A second sync, with the receiver now holding every digest, builds a batch whose payload section is empty — the have-already dividend stated in bytes. The unwelcome paths refuse: a bad batch signature, a tampered payload caught by digest while the manifest signature still verifies, and an unknown Tilak, each refusing whole.
