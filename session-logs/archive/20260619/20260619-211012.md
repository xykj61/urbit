## Decisions

- **`rish-first, sh-fallback`** named as the strategy. We don't eliminate `.sh` files; we write the `.rish` version once capability allows, and the `.sh` serves environments where Rishi isn't built yet.
- **`additive-gate.rish` waits for stream processing.** The additive gate's core logic is line-by-line filtering of `git diff` output — this requires iterating over lines in Rishi, which is not yet possible without calling `awk` or `sh`. Naming it explicitly keeps the goal visible.
- **No `mktemp` in selftest.rish.** A fixed path avoids the trailing-newline problem (Rishi has no `trim`). The shadow is rebuilt fresh on every run so determinism is preserved.
- **`sh -c` for the sed redirect.** Honest about the limitation; documented. When `write-file` lands, replace with:
  ```
  let tampered = run ["sed" "s/0x06, 24/0x07, 24/g" "${rye_lib}/std/crypto/sha3.zig"]
  write-file "${shadow}/std/crypto/sha3.zig" tampered.out
  ```
