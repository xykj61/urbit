## Decisions

- **Single-file architecture.** `rye build` only bridges one file; all Rye programs are single-file by this constraint. The logical sections are delimited by comment headers that name the future module boundaries.
- **LCS diff.** Simple and correct over fast and complex for the seed. The patience diff is a later upgrade; LCS is provably correct and readable.
- **`std.mem.trim` over `trimRight`.** Rye's std only has `trim`; for HEAD (a hex string + `\n` with no leading whitespace) the behavior is identical.
- **`std.ArrayListUnmanaged(T).empty`** for zero initialization — `.{}` fails because the struct fields have no defaults.
