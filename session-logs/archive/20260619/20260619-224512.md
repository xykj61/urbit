## Decisions

- **Self-spawning binary**: rather than two separate programs (supervisor + child), one binary serves both roles. `args[1] == "child"` switches to child mode. This keeps the seed self-contained — one file, one build, one test.
- **Deterministic test**: the supervisor knows exactly how many failures to expect. The postcondition `restarts == planned_failures` proves the loop is deterministic, not a retry with hope.
- **Not in the parity gate**: the Caravan seed spawns processes — it is effectful, not a pure computation. It is self-testing: exit code 0 means the assertions passed.
