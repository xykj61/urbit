# Session log — 2026-06-19 (Zed instant exit after Landlock)

**Stamp:** `20260619.090000`
**Editor:** Claude Code (Zed) · **Model:** Claude Opus
**Prompt:** Zed tried opening but immediately crashed out after `Landlock: fully enforced`.

## Thinking trace

1. Screenshot shows ai-jail reached Landlock enforcement, then the shell prompt returned with no Zed window — the child process exited quickly rather than a launch-script error.
2. Checked `.zed-state/data/zed/logs/telemetry.log`: Zed did briefly run (`App First Opened`, theme/keymap settings) then died — consistent with graphics init failure, not a missing binary.
3. `tools/enclosure.conf` had `USE_GPU=false`. ai-jail only grants `/dev/dri` rw when `--gpu` is passed; Zed on GNOME Wayland uses Vulkan/WebGPU and exits silently without DRI access.
4. `launch-zed.sh` defaulted `USE_GPU` to false when unset; inverted to default true for the Zed launcher only (Cursor launcher unchanged).
5. Added `libexec/zed-editor` as first-class resolve candidate (direct binary vs wrapper).
6. Documented symptom in `context/specs/enclosure-editors.md` troubleshooting table.

## Decisions

- Zed launcher defaults GPU on; user can set `USE_GPU=false` for software path with a stderr warning on Wayland.
- Personal `enclosure.conf` flipped to `USE_GPU=true` for immediate retry.
