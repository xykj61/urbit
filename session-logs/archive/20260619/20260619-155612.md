## Thinking trace

1. Framed the POSIX + TAME question precisely: POSIX is a family of interfaces, not a single design. Parts align with TAME (explicit buffers, bounded lengths, the *at syscall family, mmap with stated size, no hidden runtime); parts do not (errno global, null-terminated C strings, signal handlers, fork in multithreaded programs).

2. Identified the TAME discipline for POSIX:
   - Wrap errno at the call site, convert to Rye error unions; errno never escapes.
   - Validate paths as Zig slices with explicit lengths before converting to C strings.
   - Prefer *at variants (openat, mkdirat, etc.) for path traversal safety.
   - Avoid signal handlers where possible; use signalfd (Linux) or kqueue (BSD) instead.
   - For fork: use it only before threads start, or use posix_spawn; never fork a multithreaded process.

3. Read River's design:
   - A Wayland compositor in Zig, speaking POSIX for every display interaction.
   - The connection pattern: socket(AF_UNIX, SOCK_STREAM, 0) → connect → sendmsg/recvmsg with SCM_RIGHTS (fd-passing for SHM pool).
   - The frame pattern: mmap of a stated-length SHM pool → write pixels → notify compositor via wl_surface.commit.
   - River shows these patterns in Zig's typed idiom — no errno escape, no global state, fd-passing as a first-class operation.

4. Read Ghostty's design:
   - GPU-accelerated terminal in Zig: Wayland + X11 on Linux, Metal on macOS, WebGL in browser.
   - The backend seam: one interface (create surface, resize, draw frame, handle input) with multiple backends — exactly Brushstroke's seam design.
   - The terminal cell model: grid of (glyph, color) cells, indexed by row/column, bounded by stated grid dimensions.
   - libghostty C API: embeddable terminal surface → the "thin costume first" option for Brushstroke's text area.

5. Wrote `external-research/977_posix_and_tame_style.md`: six sections covering what POSIX gets right (explicit buffers, the *at family, mmap, no hidden runtime), where it diverges (errno, C strings, signal handlers, fork), the wrapping discipline (boundary assertions, error conversion), River as the Wayland reference, Ghostty as the Brushstroke thin costume, and the design implications.

6. Key design conclusion: POSIX is TAME-compatible (not TAME-native). The boundary wrapper is the TAME move. Ghostty's libghostty is the Brushstroke seed's thin costume. River is the Pond compositor reference for the full OS.
