## Part One — What We Mean by Itinerary

### The problem the itinerary solves

An autonomous agent with repository access can always find *something* to do. The tree is large; curiosity is cheap; and an hour of confident motion without a ruling is how policy drifts, names appear unbidden, and parity red gets explained away in prose. The **itinerary** is the positive answer: **here is the named list of stops you may execute this minute** — every precondition satisfied, every owner tag yours, every gate already open.

The vocabulary lives in LEXICON and the retirement spec:

| Term | One line |
|------|----------|
| **Gate** | A precondition owned by someone else or an unlanded event — a `(Kaeden …)` box, a door ruling, a paste not yet arrived, counsel not yet confirmed |
| **Open itinerary** | Every precondition for the next stop is satisfied **now**; the owner tag is Cursor (or the tagged autonomous sitter) |
| **Itinerary** | The **named list of stops** the sitter may execute **without** simulate/wait/ask — never an off-route license |
| **Stop** | One item on the itinerary — a witness, a wire, a doc season |
| **Smallest-first** | Order **within** the itinerary — the nearest, smallest stop wins the next leg |
| **Witness-first, red-then-green** | Method **within** the itinerary — prove on metal before claiming green; fail loudly, then land in the same sitting |
| **Journey's pause** | When the itinerary has **no further stops** — stop and summarize; a pause is a question for Kaeden, never permission to go off-route |
| **Off-route** | Work not on the itinerary — the agent names it and waits |
| **Lap** | An SLC-sized whole at a named scope — the smallest loop that closes fully within its bounds — [`20260703-033602_first-lap-goes-green.md`](20260703-033602_first-lap-goes-green.md) |

Read together: the **itinerary** is **bounded route**, not open wandering. RTAC, TASKS, and the recursion prompt hold the stops; the agent completes one stop, witnesses, commits, and returns for the next — or reaches journey's pause.

### What the itinerary is not

The itinerary is **not** the whole roadmap — ROADMAP holds the why and order across seasons. The itinerary is **this sitting's** slice.

The itinerary is **not** counsel — external-research and Claude surveys **propose**; they enter the stops only when seated or ratified.

The itinerary is **not** off-route work — *"the tree probably needs X"* waits until X is named as a stop.

The itinerary is **not** mood — *"open itinerary: RW-2 season"* is a **list**; when RW-2 is checked and parity green, journey's pause holds until Kaeden or TASKS names the next stop.

### How an itinerary moves through a sitting

A typical arc:

1. **Stops declared** in RTAC (`chrono sweep wire · RW-2 season · …`).
2. **Smallest-first** picks chrono wire (one parity block) before RW-2 (two witnesses + build).
3. **Witness-first** runs parity red if needed, then green, then commit.
4. **Itinerary refresh** — TASKS and RTAC update; crossed stops leave the list.
5. **Journey's pause** or **Kaeden word** opens the next season.

Counsel rounds (bench-as-block, crossing suite) **sit outside** the itinerary unless explicitly seated — they are Claude's lane; Cursor intakes when packets land.

## Part Two — The Crossing Manifest Seed

### Filename honesty

The artifact today is **`work-in-progress/20260703-202312_crossing_manifest_seed.rye`** — not yet `crossing_manifest.rye`. The `_seed` suffix means: **proposed, unverified, no bench claim**. A witness must run red-then-green before the name graduates. Lap one of Amphora formalizes the **manifest line**; the full vessel (zip, seal, stamp, shoulder) is larger than this seed.

### What problem it solves

Every night the Rio↔Cursor handoff improvises a container:

- Zip file, additions-only
- Parent git hash in the archive comment
- File list carried by convention and prose
- Rules re-taught in every recursion prompt

**Amphora** (vocabulary seated `20260703.203712`) names **vessel software** — preservation in motion. The nightly zip improvises an amphora; lap one formalizes the manifest line. **Amber** is **cellar software** — preservation in place; lap two pours a season into a vessel for the offsite copy. The **manifest** is the vessel's table of contents — each cargo row names **what** (Tilak mark), **proof** (SHA3-256 hex), and **path** (name in the tree). The seed parses **one row**:

```text
<mark> <sha3-256-hex> <name>
```

Example (illustrative marks):

```text
research 6047187a66560393a010859c0efc98207607a5a90a99639b0d13090e5e385330 external-research/20260703-200712_mysticetis-blocks-and-the-object-path.md
silo 36c7d27794069bf1d74d6818f4a7acd65f3def0961cd00778d5587aca401ae5c active-designing/20260703-200712_what-needs-ordering.md
log b2fb8e1bbd3110fd7c7b18ec429e978a3ab13f6331017b1c5a857a4784f02e5d session-logs/archive/20260703/20260703-200712_pottery-and-suite-reissue.md
```

The parser **refuses damage whole** — wrong field count, uppercase hex, missing name — matching the Amphora wreck rule at the line grain.

### What the seed does (mechanically)

[`crossing_manifest_seed.rye`](../work-in-progress/20260703-202312_crossing_manifest_seed.rye) exports one function's behavior via tests today:

- **`parse_manifest_line`** — split exactly three space-separated fields
- **Bounds** — mark ≤ 32 chars, name ≤ 200 chars, digest exactly 64 lowercase hex chars (SHA3-256 working tier per resins law)
- **Invariants** — TAME style: assert pre/post conditions; no silent partial parse
- **Tests** — well-formed line parses; damaged lines return `InvalidFormat`

Lap one is **parse + verify**; read file, hash bytes, build zip, and sign wait for later laps.

### Purpose in the larger crossing

```text
Shoulder line (human prose)
Parent mark (git tip)
Manifest (many lines of mark + digest + name)  ← this seed
Seal (Amber, lap two horizon)
Stamp (Kumara, horizon)
Vessel bytes (zip or amphora file)
```

Dogfood path: the loop's own nightly packet lists its cargo in manifest form so prompts stop repeating zip discipline — the vessel carries the law.
