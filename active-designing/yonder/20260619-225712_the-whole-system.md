## The One Value Model

Before any module is named, the foundation is named: **one value model**, all the way through. A value is a string, an integer, a truth, a list of values, or a record of named values — and the larger shapes built plainly from these. Values are composed side by side, never tangled. A value crosses from one module to the next without being serialized to text and reparsed on the other side — the seam never opens.

A Rishi command is a value. A Mantra commit is a value. A Brix descriptor is a value. A Pond policy is a value. A Comlink message is a value. A Brushstroke frame is computed as a pure function of values. A Tally garden is bounded by a value. A Caravan capability names a value. The coherence of the system rests on this: every module speaks the same language, and a value at the hand is the same value at the wire, at the store, at the screen.

## The Modules, Named and Composed

### Rye — The Language

Rye is the language the whole family is written in. A derivative of Zig 0.16.0 that grows into its own: explicitly sized types, assertions as first-class design, bounded gardens over region allocators, accretion over breakage, and cohesion with the open grain of RISC-V. Correctness by construction — the dangerous states are the ones a person cannot write (`990`).

Rye compiles to x86_64 today and RISC-V as the horizon target. The toolchain already demonstrates both. Every module below is a Rye program.

### Aurora — The Boot

Aurora is the boot: a relay of named, verified values from the owner key through privilege stages on RISC-V. Each stage is content-addressed, proven before execution, and hands control forward only after shape-casting the next stage's bytes against a declared type. Aurora gives way to Caravan once the verified system is ready (`991`).

On x86_64, Aurora's relay pattern maps to the privilege model (rings rather than machine/supervisor modes), with a firmware interface as the entry point. The Rye code is the same; the privilege boundary differs.

### Caravan — The Kernel

Caravan is a microkernel (`984`). The trusted core holds three responsibilities:

1. **Capability-based address spaces** — every process holds capabilities that name what it may touch. A capability is a value: it can be passed through IPC, so delegation is a message.
2. **Scheduling** — bounded time slices, priority declared at creation, preemption when the slice expires.
3. **Inter-process communication** — call (synchronous send-and-wait) and notify (asynchronous signal). The message is small; bulk data travels through Tally-bounded shared regions.

Everything else — file access, device drivers, the network stack, the display surface — runs in isolated user-space processes supervised by Caravan's process manager, the first process the kernel starts. The supervision discipline — one parent per child, restart on fall, chain-loading at startup, readiness as a signal — is proven in the Caravan seed (`caravan/seed.rye`) and grows by accretion toward the full tree.

**Five kernel invariants**, stated before any kernel code is written:
1. Isolation: no process accesses memory outside its own gardens and shared-region capabilities
2. Capability integrity: capabilities cannot be forged or widened
3. Bounded scheduling: every process receives at most its stated time slice
4. IPC safety: messages carry at most a fixed number of words and capability transfers
5. Restart guarantee: when a supervised process exits, its supervisor is notified

### Tally — The Memory

Tally is the bounded garden allocator. A garden is a named region with stated edges: allocate within the budget, fail with `error.OutOfBounds` past the budget, clear whole in one gesture when the season ends. No fragmentation, no leak, no growth without a stated limit.

Three named gardens for the desktop stack: `blob` (working memory per Mantra command), `diff` (line buffer per computation), `frame` (Brushstroke widget layout per redraw — immediate mode made tangible). Each declared at construction, cleared at the season's end.

For the microkernel: Caravan issues capabilities to Tally-bounded regions. The IPC message carries only a capability reference and length; the data stays in the shared region. Zero-copy by construction.

### Brix — The Composing Language

Brix declares what a system is made of. A `.brix` file names the project and the bricks (source files) it tracks — plain key-value, one field per line, readable by hand. The seed format is a descriptor; the full Brix grows toward a lawful composition combinator where the merge of two descriptors respects identity and associativity.

Brix is to our system what content-addressed reproducible composition is to a build: same inputs, same output, every time. The descriptor feeds Caravan's supervisor — Brix declares what services to start and what capabilities each receives; Caravan enforces what the declaration named. Declaration meets supervision; each stays simple.

### Tablecloth — The Database

Tablecloth is the general-purpose database and object storage module — our answer to what MySQL, Postgres, SQLite, and Turbopuffer provide. It stores structured data, serves queries, and holds objects for applications. Tablecloth is NOT Brix's build store; Brix has its own content-addressed artifact cache (currently inside Mantra's `.mantra/blobs/`, the same way Nix has `/nix/store`). Tablecloth serves applications; Brix serves the system composition.

Tablecloth's design inherits the TAME-aligned principles from our datastore research (`975`): write-once content-addressed identity, hard stated bounds, single-writer ACID, MVCC reads, and a narrow API surface. Its scope is broader than a build store — it is the database layer Pond applications use to persist and query data.

### Mantra — The Version Control

Mantra holds every line ever written. The weave data structure — inherited from the CRDT weave model — carries the full history of a file: each line with a stable position, a generation count, and a presence flag. The weave never shrinks; it only accretes. Merges always succeed — two states produce one deterministic result by the structure's own law. Commutativity and associativity hold: `merge(A, B)` and `merge(B, A)` converge to the same state, and three branches merged in any order arrive at the same result.

A commit-chain model links weave snapshots: each commit blob names its parent and the weave state, both by content hash. `mantra add` weaves new content; `mantra log` walks the chain; `mantra status` shows what changed. The `.brix` descriptor names the tracked files; `mantra add` without arguments walks them all.

### Rishi — The Shell

Rishi is the hand where a person meets the machine. Terse for the hand, typed for safety, a sibling of Rye that shares its value model. Values at the shell are the same values in the program: a string, an integer, a list, a record — never flattened to text and reparsed. The shell and the language share one value model, so a value crosses between them without a seam.

Rishi's capabilities: `let` bindings, string interpolation, `say`, comparison, `contains`, `map`/`where`, `run` (returning a structured record of status/out/err), `assert` as a gate, integer arithmetic, file I/O (`read-file`, `write-file`, `list-dir`), and parenthesized grouping. The parity gate runs in Rishi — the child keeps watch over the parent (`989`, `996`).

### Pond — The Enclosure

Pond is Caravan's isolation composed with Tally's bounds, expressed as a policy that is a value. An application runs inside a Pond enclosure: isolated, bounded, policy-as-a-value. The policy names what may be read, what may be written, whether the network is reachable, the memory budget, and the process limit. Everything not named in the policy is denied by the absence of a capability.

Pond gains a drawing surface through Brushstroke: an enclosure may now draw. The policy names what may be drawn to, the same way it names what paths may be read. The enclosure gains reach without losing safety.

The sandboxing discipline — namespace isolation, filesystem rules, resource limits — runs hosted today (using host OS primitives as a stand-in) and bare metal tomorrow (through Caravan's capability model). The portable execution path compiles Rye to a portable form that runs on any willing host surface (`986`).

### Brushstroke — The Display Surface

Brushstroke is the family's drawing layer. Immediate mode: an interface redrawn from current values every frame, holding no retained tree that can drift from the truth. One seam abstracts multiple backends:

- **Native backend** — direct hardware rendering on x86_64. A Wayland compositor manages the window; a GPU pipeline draws the frame. The compositor provides the surface; Brushstroke fills it.
- **Portable backend** — compile Rye to a portable executable form that runs through a host's presentation pipeline. The same frame-garden discipline, the same widget seam, a different surface below.
- **Bare-metal backend** — on RISC-V, Brushstroke draws directly to the framebuffer through Aurora's device access. The display driver runs as a Caravan-supervised service, restarted on fall.

The widget vocabulary grows as needed: text, box, input, list. The `frame` garden in Tally is Brushstroke's memory: every redraw is a season — opened at the start of `render`, filled with the widget tree computed from values, cleared whole when the frame is submitted.

**Skate** is the terminal surface — a GPU-accelerated text area inside a Brushstroke window, where Rishi speaks and Mantra's output lands. Named for the joy of skating through a terminal. Skate is written from scratch in Rye; its backend seam and text rendering are our own, informed by public documentation and protocol specs rather than by any GPL implementation.

### Comlink — The Network

Comlink carries values between identities. A message is typed, content-addressed (SHA3 hash), sealed at identity (owner keys), and validated where it lands. Exactly-once delivery follows from content-naming: a name is a promise; the bytes behind it will never change. Security attaches to the content, not the channel (`993`).

The sealed datagram across two harts (proven in Aurora) is the first wire. The device wire (emulated virtio-net between two QEMU machines) is the next rung. The full Comlink grows from these toward encrypted, identity-bound message delivery across a bounded network.

## How the Modules Compose

```
[The Whole System]

  Aurora (boot) → Caravan (kernel) → Pond (enclosure)
                      │                    │
                      ├── Tally (memory)   ├── Rishi (shell)
                      ├── Brix (compose)   ├── Mantra (history)
                      ├── Tablecloth (database)  ├── Brushstroke (surface)
                      └── Comlink (net)    └── (applications)

  ← All written in Rye, all sharing one value model →
  ← All governed by TAME: safety, performance, joy →
```

Aurora wakes the machine and loads Caravan. Caravan starts its supervision tree: VFS, network (Comlink), display (Brushstroke's driver), and Pond enclosures. Each service receives capabilities issued by Caravan, bounded by Tally, declared by Brix. Inside a Pond enclosure, a person runs Rishi, types Mantra commands, and sees the result drawn by Brushstroke — every value the same value, every seam closed.

## Two Platform Tracks

**x86_64 now (hosted, then bare metal):**

Every Rye program already compiles to x86_64. A hosted Pond runs on the current machine, using host OS primitives for isolation. A native Wayland compositor (studied from the display-layer sources in `gratitude/`) manages the window. Bare-metal x86_64 follows: Aurora's relay maps to x86_64 privilege rings; Caravan runs the same supervision tree.

**RISC-V as the horizon:**

Aurora is designed in deep cohesion with RISC-V (`991`). The Rye code is the same; the privilege boundary differs. When Aurora's relay wakes Caravan on RISC-V, the same desktop application runs on open silicon.

## The Invariants of the Whole

Stated before the whole is built, in the TAME order:

1. **One value model throughout.** A value at the shell is the same value at the store, at the wire, at the screen. No serialization seam opens between modules.
2. **Bounded everything.** Every allocation, every collection, every IPC message, every time slice names its maximum. A request past the budget fails with a named error, not a silent growth.
3. **Safety by construction.** The dangerous states are the ones a person cannot write. The kernel enforces isolation through hardware. The language enforces types through the compiler. The allocator enforces bounds through Tally. Each layer adds a boundary the next layer cannot cross.
4. **Accretion, never breakage.** A name once given is a promise. A version once published does not change. The system grows by adding, never by revising what it already holds.
5. **Grown from what already works.** No module is designed whole from scratch. Each grows from a seed that runs, by degrees, behind the gates.
