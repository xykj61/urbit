## A System Is Not a Program

A programming language gives one process a world: memory, types, calls, coordination inside one address space. A **system** is an ensemble of programs — no global supervisor, no shared garbage collector, no one in charge. At that scale the question shifts from "how do I compute?" to "how do pieces **connect**?"

Our answer: they connect by **values** flowing through **simple services**, each service holding one part of the flow.

---

## Four Operations, Kept Apart

Every cross-module interaction is one of four operations. Mixing them braids concerns; keeping them separate keeps each single-stranded (`995`).

| Operation | Meaning | Our modules |
|-----------|---------|-------------|
| **Transform** | Compute a new value from inputs | Rye programs, Rishi scripts, Mantra weave/diff, Lattice matmul, Scribble parse |
| **Move** | Carry a value across a boundary | Comlink datagrams, Caravan IPC (future), Brushstroke frame submit |
| **Route** | Choose where a value goes next | Brix descriptors, Caravan supervision tree, Pond policy, Comlink naming |
| **Remember** | Persist a value under an enduring name | Tablecloth blobs, Mantra commits, `.brix` + HEAD chain, Aurora stage hashes |

A module may transform and remember (Mantra), yet it should not also route network traffic in the same breath — that is Comlink's job. Pond **routes** capabilities; it does not **transform** application logic.

## Simple Services

Each module is a **simple service**: one primary job, a tiny surface, a few verbs.

| Module | One job | Primary verbs |
|--------|---------|---------------|
| **Rye** | Execute typed programs | compile, run |
| **Rishi** | The hand at the prompt | let, say, run, assert |
| **Brix** | Declare what to build | describe bricks, merge descriptors |
| **Tablecloth** | Hold artifacts by hash | put, get |
| **Mantra** | Remember every line | init, add, status, log |
| **Tally** | Bound memory in gardens | alloc, clear |
| **Caravan** | Watch and restart | supervise, restart |
| **Aurora** | Wake the machine | relay, verify, hand off |
| **Pond** | Enclose by policy | open, deny |
| **Comlink** | Move sealed values | seal, send, receive |
| **Brushstroke** | Draw from values | redraw |
| **Skate** | Render a text grid | present cells |
| **Scribble** | Parse prose to values | parse blocks |

When a service grows a second job, we split it — or compose another service beside it.

## Values With Enduring Names

What must persist receives a **name that endures** — ideally a content hash, so the name is a promise about exact bytes.

- **Tablecloth** — SHA3-256 of the blob
- **Mantra** — commit chain + weave snapshots
- **Brix** — descriptor version stamps; future derivations named by input hash
- **Aurora** — each boot stage image named by its contents
- **Comlink** — message identity is the hash of what it says

Accretion follows: new names appear; old names keep meaning (`990`). A rename is a removal wearing a costume — we add beside, never erase.

## The Systems Failure Model

In any real system, **partial failure is normal**. Networks partition, processes fall, disks stall. The language inside one program can pretend otherwise; the system cannot.

Our responses, stated as design law:

1. **Bound every wait** — timeouts on IPC, network, and supervision handshakes.
2. **Idempotent receives** — Comlink delivers exactly-once by content-name; a duplicate is a name already seen.
3. **Restart on fall** — Caravan supervises; a crashed service returns clean, garden cleared.
4. **Fail named** — Tally returns `OutOfBounds`; Rishi `assert` stops with a reason; gates turn RED on divergence.

Partial failure is not an exception path — it is the path the design assumes from the first line.

---

## One Value Model, Closed Seams

The four operations only work if a value at the hand is the same value at the wire, the store, and the screen (`983`). Serialization opens a seam; we keep seams closed: Rishi records, Mantra commits, Brix descriptors, Pond policies, Brushstroke frames, Comlink payloads — all the same shapes.

Correctness by construction (`990`) makes dangerous states unwritable inside each service. Flow of values makes dangerous *compositions* visible: if a module must marshal to text to talk to its neighbor, the braid is showing — split the service.

## How the Map Composes

```
Transform (Rye, Rishi, Mantra, Scribble, Lattice)
    ↓ values
Remember (Tablecloth, Mantra, Aurora stages)
    ↓ named artifacts
Route (Brix, Caravan, Pond)
    ↓ capabilities + policy
Move (Comlink, Brushstroke redraw, Skate present)
    ↓ across boundary
Transform again …
```

Aurora wakes the machine; Caravan holds the tree; Tally bounds every garden; Brix declares the course; the person meets Rishi; Brushstroke and Skate show the result.

---

## The Smallest Form

The flow begins as one sealed, content-named message between two harts (running). From that single **move** with **remember** by hash, the whole network grows (`993`). The display flow begins as one **frame** drawn from values (`brushstroke/seed.rye`). Gall's law applies to the flow itself: prove one operation honest before composing the next.

---

*May every value know whether it is transformed, moved, routed, or remembered. May each service stay about one thing. May partial failure find us ready — bounded, named, and able to restart. And may the whole system speak one language of values, flowing without a braid.*
