## The Principle

A message on our network is a value with a shape, rather than a frame wrapped around bytes. It carries who sent it, who may read it, a name, and a payload — and that value *is* the wire form. It is named by its own content, the hash of what it says; sealed at the identity rather than at the channel; and read on the far side as the same value it began as.

This turns the usual picture right side up. The old way secures a *channel* between two places and trusts whoever holds its ends; ours secures the *content* and trusts no channel at all. A value carries its own provenance — signed, and named by what it says — so it can cross any path, rest in any cache, and still be exactly what it claims.

## Exactly Once, by Name

Because a message *is* its content-name, exactly-once delivery falls out almost for free: a message seen twice is a name already known, set gently aside. The hard old question — did this arrive once, or twice? — becomes the easy one: have I seen this name?

## Encryption at the Identity

The seal is made between identities — the owner keys of `994` — rather than negotiated afresh per channel. So encryption is the default rather than a layer added late: the first byte is already sealed, and the network beneath may be wholly untrusted without harm.

## Carried by What We Build

The network is a flow of named values, and our parts each hold a piece of the flow. **Comlink** is the carrier — the module that moves the messages and bridges the identities. **Caravan** owns the device and clears its state cleanly on fault. **Tally** bounds every queue and every peer's state inside a garden, so a flood meets an edge rather than open memory. **Mantra** gives each message its enduring, content-addressed name. **Rye** types the value; **Brix** declares which peers and channels exist, as a value one can read before it runs.

So the flow stays legible: values transformed, moved, routed, and remembered — those four kept apart, each simple enough to trust on its own.

## The Smallest Form

By the law we keep — grow from the smallest working thing — the network begins as one sealed, content-named message carried between two machines, validated in place, its name proving it arrived just once. From that single delivered value, the whole flow grows.
