## What Aurora Is

Aurora is the boot — the first light between a cold processor and a system that lives. It is built for RISC-V first, so the machine that wakes and the language that wakes it, Rye, cohere down to the instruction. Where most boots are a tangle of mutable state passed hand to hand on trust, Aurora is something quieter and more legible: a flow of named values, each stage a value handed to the next, from a root that cannot be forged to a system that can be trusted.

## Rooted in the Owner Key

A boot must begin somewhere it can trust absolutely, and Aurora begins at the owner key — the single root of `994`, the identity a person keeps. From the very first instruction, the machine's right to wake is the owner's right; nothing runs that the root has not vouched for. Trust is *proven low*, at the root, rather than assumed high and late, after the machine has already run code it never checked.

## A Relay of Named, Verified Values

Aurora is a relay. Control passes forward in stages, and three things hold at every handoff.

- **Each stage is a value.** A value with a shape — what was found, what was decided, what comes next — handed to the stage that follows, rather than a half-configured machine passed along.
- **Each stage is named by its content.** Mantra gives every stage an enduring, content-addressed name, so a finished boot can say *exactly* what it ran, stage by stage. A boot becomes a thing you can read back, rather than a dark passage you must trust blind.
- **Each stage proves the next before trusting it.** Before control is handed forward, the next stage is shape-cast (`992`): its bytes are proven to be the verified thing they claim before they are ever set running. Privilege flows onward only through a door that has been checked.

This is the privilege relay made honest: control descends from the machine's most privileged mode toward the living system, and at each step the handoff is a named value whose shape is proven.

## Bounded, Owned, Declared

Aurora leans on the parts we are already growing. **Tally** bounds each stage's memory in a garden, cleared whole when the stage's work is done, so a boot never sprawls. **Brix** declares the stages and their order as a value one can read before the machine runs — the boot described, rather than configured by mutable command. And **Caravan** is where Aurora is headed: the boot's last act is to hand a verified, named, living system to the supervisor that will keep it.

## Grown from the Seed

By the law we keep, Aurora grows from the smallest boot that works, never designed whole. That smallest boot already runs: a single hart wakes on a freestanding RISC-V machine, gives itself a stack, speaks one asserted line, and rests — the seed. From it grew the first relay: three stages that hand a value along a line, each handoff a tail call, each stage asserting what it leans on. The next growth is a stage that hands the next a value *it chose* — a real decision, the seed of a boot that selects what runs next. Each stage is added only once it runs, behind the gates, so the boot is always a living thing being grown, rather than a grand design awaiting its first breath.

## Where Aurora Ends

Aurora knows where it stops. Its work is the dawn — to carry the machine from cold silicon to a verified, named, living system — and then to hand that system to Caravan and step back. The boot gives way to the day. What Aurora leaves behind is more than a machine merely running: it is a machine that can say, in named and verified stages, exactly how it came to be running — owned from the first instruction by the one who holds the root.
