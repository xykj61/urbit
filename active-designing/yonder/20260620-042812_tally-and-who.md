## Our Own Ground

This is a clean room. We name only what is ours — **Tally**, **Rye**, **Caravan**, **Comlink**, **Aurora**, **Pond**, **Mantra**, **Brushstroke**, **Rishi** — together with **RISC-V** and **x86_64**. **Tally** is the module that **bounds memory in gardens** — a `Region` over caller-provided storage, bump allocate within, `error.OutOfBounds` past the edge, `clear` whole (`tally/seed.rye`, `tally/gardens.rye`). It is running today; freestanding harts still use fixed constants where Tally will later name the same edges.

## Tally Is Not a Who

| Concept | Kind | Question it answers |
|---------|------|---------------------|
| **Executor, sealer, opener, router** | *Who* (`972`) | Which role holds **agency** at a seam? |
| **Datagram** | *What* (`973`) | Which **value layout** crosses the seam? |
| **Tally garden** | *Where* | Which **bounded memory** may this role use **right now**? |

Tally does not run code, sign names, decrypt, or choose the next hop. It **limits and releases the ground** each who stands on while doing those jobs. Confusing Tally with a who would smuggle memory into agency — we keep them apart (`995`).

## Tally Meets Each Who

### Executor — one garden (or several) per runner

Every **executor** needs a place to work:

| Executor | Tally today / horizon | Season end |
|----------|----------------------|------------|
| **Hart** | Stack carved by `mhartid × 64 KiB` (`974`); mailbox at fixed addresses | Hart runs until halt/`wfi`; horizon: named stack garden per hart |
| **Process** | `init.garden` for the process season (`990`); Comlink binaries allocate there | Process exit clears the garden |
| **Supervised child** | `caravan/bounded.rye` — child in a stated byte budget | Parent restarts child; garden cleared on fall |

**Caravan + Tally** compose: the parent is executor-of-supervision; the child is executor-of-work inside a **Tally-bounded region**. When the child exceeds budget, it falls; the supervisor restarts — the garden clears whole, the same move Caravan and Tally were designed to make together (`983`).

Executors **transform** values inside gardens (`977`). Seal and open in `posted.rye` use stack arrays today; Tally makes those edges explicit and reusable across seeds.

### Sealer — public material only on the wire; secrets stay in gardens

The **sealer** is identity on the datagram — public key, signature (`973`). Private signing material lives in the **executor's garden** during the transform step:

- Alice's secret never enters the datagram (*what*)
- While hart 0 seals, key material and scratch buffers belong to **hart 0's memory region**
- Tally states: no unbounded scratch; no alloc past edge while building ciphertext and name

Tally does not replace Ed25519; it **bounds the memory the sealer's executor may touch** while sealing.

### Opener — decrypt into a bounded buffer, then clear

The **opener** holds the recipient secret off the wire (`972`). On receive:

1. Shape-cast the datagram in a **fixed wire buffer** (`wire_capacity`)
2. Derive shared secret; decrypt into an **opened buffer** in the opener's garden
3. Use plaintext; then **clear** the season when the handler completes

Bob's private key and opened message live in **hart 1's region**, not in shared mailbox RAM the sender still sees. Tally's discipline rhymes with **opener ≠ sealer**: public attestation on the wire; private material in the opener's garden.

Future borrow checking (`990`, `983`): a reference may not outlive the garden that holds the opened plaintext — Tally supplies the **when it dies**; the checker supplies **who may reference until then**.

### Router — queues and peer state stay bounded

The **router** forwards *what* (often only content-names) without opening (`993`). Routers still need memory:

- Pending datagram queue depth
- Seen-name set for exactly-once dedup
- Per-peer handshake state

Each structure is a **named garden** in horizon Comlink — `relay`, `seen`, `peer` — with stated maximums. A flood meets `OutOfBounds` at the edge, not silent heap growth (`973`, `975` virtio queues likewise).

**Pond** routes by policy; **Tally** enforces `mem_mb` on what the enclosed executor may allocate (`986`). Route is Caravan/Pond; the cap is Tally.

## Tally Meets the What — Datagram Buffers

The **datagram** is layout, not memory policy. Tally wraps the **containers** it passes through:

| Buffer role | Typical bound | Module |
|-------------|---------------|--------|
| Wire serialization | `512` bytes (`wire_capacity`) | Comlink / Aurora posted |
| RX socket buffer | One datagram per recv | `hosted_wire.rye` |
| Virtio descriptor chain | Queue depth × max frame | `975` |
| Shared mailbox | `len` + payload under capacity | `posted.rye` |

Shape-cast (`992`) proves the bytes match the type; Tally proves the **container** was never overrun while those bytes were assembled or copied. **What** is validated by crypto and offsets; **where** is validated by Tally.

## Named Gardens — Who Uses Which Name

`tally/gardens.rye` names regions for the desktop stack; the same pattern maps to `972`:

| Garden name | Primary who / what | Job |
|-------------|-------------------|-----|
| **blob** | Mantra transform | Serialize weave, read buffers |
| **diff** | Mantra transform | LCS working space |
| **frame** | Brushstroke executor | Widget tree per redraw; season = one frame |
| **wire** (horizon) | Comlink move | Assemble or hold one datagram |
| **stack** (horizon) | Hart / process executor | Per-runner working stack |
| **seen** (horizon) | Router | Content-names already delivered |

Naming gardens is how **one executor** holds several bounded seasons without braiding them — blob work does not eat frame memory (`977` simple services).

## init.garden and Tally — Cousins, Not Duplicates

Rye's **`init.garden`** is the process **season allocator** — allocate freely for the session, clear on exit (`990`). Tally **`Region`** is the module pattern: explicit edge, bump pointer, clear whole on demand.

| | **init.garden** | **Tally Region** |
|---|-----------------|------------------|
| **Scope** | Whole hosted process | Caller-chosen slice (per service, per frame) |
| **Edge** | Implicit (OS memory) | Stated at `init(buf)` |
| **Clear** | Exit | `clear()` anytime — restart, end of frame, after open |
| **Who uses** | Rye, Rishi, rye build | Caravan children, Mantra, Brushstroke, future Comlink |

Pond will measure **Tally** gardens inside an enclosure; the hand may still use **init.garden** at the shell. Both honor the same metaphor; Tally makes the edge **visible in code** for modules that must prove bounds to the gates.

## Failure — Clear Whole When a Who Stops

When an executor faults or a Caravan child falls:

1. Caravan **routes** restart (`972` router at supervision layer)
2. Tally **clears** the child's garden whole
3. No dangling references into freed memory — partner to borrow-checker horizon (`983`)

When a relay dedups a duplicate datagram, the router **sets aside** the *what* by name; Tally does not grow the seen-set past its garden. Partial failure stays normal (`977`).

## Caveats

- **Tally does not authenticate.** Gardens bound size; crypto bounds trust (`973`).
- **Shared mailbox RAM** between harts is not yet a Tally region in code — fences and fixed addresses carry the proof; Tally names the horizon.
- **Do not call a garden a who.** A garden is where Bob's secret lives; Bob-as-opener is the role.

---

## Conclusion

`972` names **who acts** and **what moves**. **Tally names where they may work** — per executor, per transform, per frame, per queue — with edges stated and seasons ended in one gesture. Executors run in gardens; sealers and openers touch secrets only inside their executor's bounds; routers hold pending *whats* in bounded queues. The datagram stays a layout; Tally stays the floor beneath every role at the seam.
