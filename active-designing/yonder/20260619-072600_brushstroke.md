## Mission

To give Rye OS a surface — a way for the system to show itself to a person, drawn from the same values that flow everywhere else within it. **Brushstroke** is that surface, and its first task is to render our own documentation, so the system can at last be walked through rather than only read folder by folder.

## Problem

We document richly, yet a reader meets the work as a stack of plain-text files with no place to move through it — no front door, no following of a link, no sense of the whole's shape. And we have, so far, no way at all to draw an interface from our values: every part of the family computes, names, seals, and remembers values, yet none of them lets a person *see* one. A system that cannot show itself stays a library only the patient can enter.

## Opportunity

The opportunity is a kind of interface that fits our grain exactly. An **immediate-mode** surface holds no second copy of the truth — no retained tree of widgets living between frames that can quietly disagree with the values beneath it. Instead it redraws from the current values each frame: the interface is a pure function of the state, computed fresh. This is the same shape the rest of the family already keeps — values flow, and what you see is computed from them, with nothing stale persisting in between. A retained tree is a second copy, and a second copy can drift; immediate-mode keeps one copy and draws it. That is simplicity in its strict sense, single-stranded, and it is the reason this kind of surface, and no other, belongs here.

A second gift compounds it. A document, in our world, is a named, content-addressed value — Mantra's domain, the same kind of thing as a message, a configuration, a value at Rishi's prompt: named, bounded, owned. If documents are values and Brushstroke draws values, then the documentation surface is simply values flowing to the eye — content addressed by its name and rendered by a pure function of itself, rather than text run through a templating machine. The surface becomes the clearest demonstration of the one value model there could be: the system shown in its own grain.

## Architecture

Brushstroke is three simple parts, composed:

- **The redraw.** A pure pass from the current values to a frame, run anew each time something changes. It retains nothing it must keep in step with the truth; the values are the truth, and the frame is their shadow.
- **The seam.** One backend abstraction that decouples *what to draw* from *where it lands*. A single Brushstroke program paints through this one seam to whatever surface is present — a drawing surface within a person's reach today, a native window tomorrow — without the program knowing or caring which. One interface, many surfaces, one clean joint between them.
- **The vocabulary.** A small set of things to draw — text, a box, a button, a list, a link — grown one at a time, each proven, none raised before the work needs it.

To reach a person at a distance, Brushstroke compiles to a **portable executable form**: a small, sandboxed artifact that runs wherever a willing host receives it, drawing through the host's own surface. This is a target distinct from the bare machine Aurora wakes — there, Brushstroke would paint to a native surface directly — yet both rest on the same redraw and the same seam. And throughout, Brushstroke is written in Rye, on Rye's own standard library, and cut to the open machine (RISC-V) as everything here is.

## Caveats

- **The documents come first; the surface second.** The reference and the map are plain documents we can write today, and they must not wait on Brushstroke. A surface that renders nothing is scaffolding, and we keep no debt.
- **The redraw is not free.** Drawing the whole frame each time it changes is more work than mending a retained tree in place. For a surface that changes only when a person acts, that cost is small and the simplicity it buys is large; we spend it knowingly, and only where the bargain holds.
- **The portable form is its own discipline.** A small artifact, a clean bridge to the host's surface, input carried in and pixels carried out — these are real, and we meet them one widget at a time rather than all at once.
- **Grown, never bolted on.** Brushstroke begins as the smallest thing that draws and becomes the larger surface by degrees, each stage running before the next is added — the law we hold for every module.

## Longer Horizon

Further out, Brushstroke draws more than documentation. Rishi composes a view and hands it to the surface; a person reads the whole system — its modules, its history in Mantra, its open threads — by moving through a drawn place rather than walking the files. The surface reaches native windows as readily as the portable form, through the one seam. And the deepest turn: the system shows itself in its own hand, every page a value it computed, drawn by a surface it wrote. The documentation site is the first citizen of that horizon, and the proof that the value model reaches all the way to the eye.

## Conclusion

Brushstroke is a surface that draws itself from values — immediate-mode at its core, one seam to many surfaces, a small vocabulary grown as needed — beginning with our own documentation and reaching, in time, to show the whole system in its own grain. It asks for nothing exotic and nothing retained: only that what a person sees be computed, fresh and honest, from the values that were already flowing. We begin, as always, with the smallest thing that runs — one true line, drawn to a surface.

## Display Seam (native x86_64)

The native backend crosses the OS boundary through three narrow wrappers (`985`, proven in `wayland_seed.rye`):

| Wrapper | Job |
|---------|-----|
| **`Display.connect`** | Open the compositor socket, hold `wl_display` for the session |
| **`Display.Surface`** | Create `wl_surface`, xdg toplevel, configure/ack, commit lifecycle |
| **`Display.Buffer`** | SHM pool from a bounded memfd, stride, attach, damage |

Brushstroke logic speaks only in `Frame` values and these wrappers — no raw fds or wire types above the seam. The hosted seed (`seed.rye`) uses stdout as the backend; the Wayland seed swaps the backend, not the redraw discipline.
