## The name

**Lotus** — the flower that rises clean from deep water. A studio that must stay pure under load: the performance path bounded, the graph of the session honest, the interface showing truth because it keeps no second copy to drift from it.

## Two lodestars, one clean room

| Source | What Lotus learns | How we hold it |
|--------|-------------------|----------------|
| **REAPER** (Cockos) | Item model, routing, ReaScript extensibility, JSFX-style live editing, small fast binary, edit-while-playing | Public docs in `gratitude/reaper/docs/`; proprietary — never link |
| **AP Mastering** (Alain Paul) | Tiered plugin UI (retro / preset / expert), meters, sidechain clarity, mastering-grade craft on screen | Public pages + YouTube; never copy assets |
| **Genesis** (Kelley) | Hot-loop discipline, plugin safety ambition, open collaborative studio | GPL-3 gitlink distance; `gratitude/genesis-daw.md` |

Lotus is **like REAPER** in workflow honesty — multitrack items, takes, nested buses, actions bound to keys, user-scriptable extensions in **Rye** rather than EEL/Lua/Python.

Lotus surfaces are **inspired by AP Mastering** — immediate connection between ear and control, pages that match skill level, expert depth without hiding the engine — drawn through **Brushstroke** immediate-mode, not a retained plugin DOM.

## Architecture sketch (horizon)

```
Weave (session facts: regions, automation, plugin state as signed events)
  → fold → plain values
  → Brushstroke (mixer, editors, plugin faces)
  → **Maitreya** (hot-loop audio path; video decode horizon — Tally-bounded buffers, Caravan-supervised hosts)
  → host media seam (ALSA/JACK/Pulse on Linux today; sovereign path later)
```

**Lotus** sits above **Maitreya** — the DAW workflow and project fold; Maitreya is the Rye OS media driver framework (audio first, video as shared seam when the lap opens). **Realidream** and **Linengrow** integrate playback through Maitreya middleware rather than ad hoc host codecs.

**Bookie** governs who may change a value (automation lane vs read-only witness). **Dexter** carries control surface, keyboard, and pointer into the fold. **Kumara** signs exports and collaborative edits into Weave when multiplayer earns its lap.

## SLC order

Lotus is **not** the live rung. The ladder stands:

1. **SLC-2a** — Skate viewer (drawn terminal) — Brushstroke proves the surface
2. **SLC-2b** — input in the window (Dexter on Wayland)
3. **Lotus seed** — transport + one track + one JSFX-shaped native effect + meter — the smallest studio that can record and play without skipping
4. Grow by degrees toward REAPER-shaped depth and AP Mastering-shaped plugin faces
