## What it is

A **witness bolt** is a **minimal, pinned catalog fixture** ‚Äî one peer, one bolt name, one revision, and a small set of named leaves ‚Äî chosen so a witness can prove a lap **in the happy zone**: fast, isolated, bounded, and repeatable on metal.

The bolt is **not** production data. It is **evidence furniture**: predictable bytes, predictable digests, predictable have-already and need-resin lanes, so a green witness means the mechanism worked ‚Äî not that a real peer synced a real tree.

Grammar uses seated namespace words:

| Field | Role in witnesses |
|-------|---------------------|
| **peer** | Names who holds the bolt (`peerB` on the wire; `local` in `recall_lap1` selftests) |
| **bolt** | The unit under test (`b` on the wire; `witness-bolt` in NS-L1) |
| **revision** | Usually **1** for wire laps; higher only when the lap tests immutability |
| **path** | Leaf names (`a.txt`, `b.txt`, `large.bin`, derived `{path}.b{N}`) |
| **resin** | Bytes at each path; **Tilak** marks layout (`plain-text`, `plain-bytes`, `bead-index`) |

A witness bolt **fits the lap it proves**. It does not try to be general.

## What it is not

- **Not a conformance suite** ‚Äî one bolt per lap, not exhaustive peer diversity.
- **Not a capacity demo by default** ‚Äî size grows only when the lap under test requires it (chunking, beading).
- **Not shared across unrelated modules** ‚Äî `recall_lap1` keeps its own `witness-bolt`; wire laps use `peerB`/`b`/`1` so guests and hosted delivery share one story without forking NS-L1.

## The closing test (grain)

A witness bolt **cuts with the grain** when:

1. **Bounded** ‚Äî leaf count and bytes stay within catalog ceilings (`max_batch_entries`, `max_resin_bytes`, wire `max_message`).
2. **Witness-first** ‚Äî parity runs green on metal; the bolt exists to be exercised, not admired.
3. **Logic before carriage** ‚Äî in-process catalog proof lands before wire proof (NS-L3 lap 3 before 3w).
4. **Whole refusal** ‚Äî bad seal, bad digest, or bad chunk assembly refuses the crossing entire; the bolt stays unchanged on failure.
5. **Referential transparency** ‚Äî after a green run, `recall` (or `recallBeaded`) on the fetcher returns the same bytes the source held.

## Variants seated on the bench

| Variant | Leaves | Proves | Where |
|---------|--------|--------|-------|
| **NS-L1 bolt** | `local` / `witness-bolt` / 1 / `greeting.txt` | Local recall, revision immutability | `mantra/recall_lap1.rye` |
| **Wire bolt (small)** | `peerB` / `b` / 1 ‚Äî `a.txt`=`alpha`, `b.txt`=`beta` | Single-frame sync and batch; alpha held, beta need-resin | Sync wire, batch wire demo |
| **Chunk bolt** | Same peer/bolt; `b.txt` = **400 B** `plain-bytes` | Multi-frame kind **0x04** chunking (> 340 B batch) | `recall_batch_delivery` chunkdemo, device guests |
| **Beaded bolt** | `large.bin` beaded + `{path}.b{N}` derived leaves | Bead-index and beads cross batch wire; `recallBeaded` on fetcher | **landed** ¬∑ hosted `beadeddemo` + device virtio lab ¬∑ ~1467 B batch |

**Signer seed** for batch wire witnesses: `0x61` repeated (Ed25519 deterministic). **Held digest** for have-already lane: SHA3-256 of `"alpha"` as 64-char hex.

## How to extend a witness bolt

When a lap needs more than the small wire bolt:

1. **Name the new constraint** in this brief or the namespace brief (capacity, beading, revision).
2. **Grow the fixture minimally** ‚Äî one new leaf or one larger resin, not a forest.
3. **Keep have-already / need-resin counts assertable** ‚Äî witnesses print and assert `had_already` and `needed`.
4. **Extend the existing `.rish` witness** when the lap is the same module; add a new witness only when the module seam is new.
