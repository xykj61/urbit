## 1. Brix and Silo (infuse config)

### The one-line answer

**The Silo module name is retired.** Configuration composition lives in **Brix bonds**; content storage lives in **Tablecloth**; lowercase **silo** names only the reading technique and astrology lane — never a Rye OS module.

### History, stated plainly

| Era | Name | Role today |
|-----|------|------------|
| Early 2026 | **Silo** module | Split: **Brix** = composer · **Tablecloth** = store (`20260620` vocabulary pass) |
| infuse.nix pattern | Nix override combinator | **Brix bond** — infuse values into a declared whole without mutating the base |
| Astrology / gratitude | **silo** (lowercase) | Clean-room reading technique — unrelated to package config |

The Open Question conflated two different "Silos": the **retired module name** and the **infuse configuration pattern**. Only the pattern survives — inside Brix, not as a sibling module.

### What we do not resurrect

- A distinct **Silo config layer** beside Brix
- `.silo` project descriptors (superseded by `.brix` + Bron values)
- Nix/infuse as a runtime dependency (LGPL; ideas only per gratitude licenses)

**Bron** carries override *values*; **Brix** applies bonds. See counsel `222500`.
