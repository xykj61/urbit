## The 0.16 Question, Answered by Measurement

**Rye already writes 0.16's new world as its native idiom** — this is the honest headline, and it deserved measuring rather than assuming. Every hosted entrypoint is `std.process.Init`; `std.Io` threads through 74 signatures; `Io.Dir` carries the file work; `.empty` decl-literals and `ArrayListUnmanaged` are the norm; the legacy writer style has zero surviving sites. The sketchbook, for contrast, lived in the old world (GPA mains, managed lists, direct arenas) — the migration Kaeden's question wondered about is not pending; it already happened, and it happened *as Rye*. What the measurement found genuinely unused, each with a real consumer already named in this tree:

**First, the one that matters: `std.process.Child` spawn-then-wait, and behind it `Io.Group`'s async and concurrent.** Rishi's `run` blocks — that single fact is the structural boundary keeping all seven wire-lab `.sh` scripts and `aurora/run.sh` alive, named as such in the seam study. The std API is not the gap (Caravan already spawns children through `std.process.Child`); the gap is Rishi the *language* having no verb for it. This bench proposes the pair fully and **parks the names**: `spawn ["cmd" ...]` returning a handle record, and `wait-for <handle>` returning the familiar `{out, err, code, ok}` — two verbs, bounded (a named max on live handles), which would let the background-orchestration family of `.sh` scripts migrate the same honest way `tame_style_scan` did, leaving only the bootstrap and external-interpreter seams, which are correct forever. New verbs are new names; the design is complete, the word is Kaeden's.

**Second, small and on-touch: labeled `continue :state` switches** — proven in the stdlib's own tokenizer, unused in this tree, and exactly the shape of Rishi's statement dispatcher and any future Bron/Tilak parser. Guidance for the next touch of those files, never a sweep.

**Explicitly not recommended:** adopting `Io.async` for its own sake anywhere the current sequential shape is already honest, or any migration whose only argument is that the capability exists. The tree's restraint here is not lag; it is the same proven-ground rule that deferred `parse_int` until fifteen real call sites existed.
