# Module Name Provenance — Dexter, Kumara, Bookie

**Language:** EN
**Last updated:** 2026-06-30
**Style:** Radiant (see `../RADIANT_STYLE.md`)

---

Kaeden asked that the reasoning behind three module names travel with the names themselves. This spec records that provenance. The names are **given**; whether each earns a full module home is still open in `work-in-progress/TASKS.md`.

## Dexter — input

Chosen for the module that carries keyboard and pointer events into the fold, on both the hosted Wayland backend and the sovereign framebuffer backend.

- **Dexter** — the right and skillful hand; dexterity as the skill of acting.
- **Texter** — an echo of one who types the system into motion.
- **Zendex** — the enterprise of **Robert Sealy Livermore** (Bob Livermore), Kaeden's maternal grandfather, who passed in 1997. The name keeps that memory woven into the thing itself rather than set apart from it.

## Kumara — identity

Chosen for the keypair primitive that signs a fact before it enters **Weave**.

- The four **Kumaras** of *Srimad Bhagavatam* — Sanaka, Sanandana, Sanatana, and **Sanat-kumara** — eternal, ever-youthful, ever-pure sage-children.
- Identity is the unchanging, trustworthy signer behind every signature; the name fits the role and honors the source.

## Bookie — the borrow discipline

Chosen for the keeper of the book of **who may change a value and who may only witness it** — many witnesses or one changer.

- Vivid as a **turf accountant** keeping the ledger straight (runners, chalkers, rafflers of the old betting houses).
- **Bookie** watches not what a value *is* (string, integer, list, record — that stays in Rye's hands) but who is **allowed to change it**.
- As a **lived discipline** it holds today in assertions and capabilities; as a checker that reads ownership from code, it waits for the AST-horizon lint tier.
