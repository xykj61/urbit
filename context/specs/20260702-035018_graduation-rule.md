## What the rule says

**An aspect becomes a module on the day its second consumer arrives from outside its host.**

That is the whole rule. Everything below explains what the words mean and how we apply it.

---

## Module and aspect, said once

A **module** is a named home: its own directory, its own invariants and budgets, its own witnesses, and a meaning a person can say in one line. A module could stand alone if the family around it fell away.

An **aspect** is a named capability that lives *inside* a module's boundary — sharing its host's budget and witnesses, carrying its own name in the lexicon and in conversation, and owning no directory of its own. Skate is an aspect of Brushstroke. The hosted wire is an aspect of Comlink. An aspect is a promise of shape without the ceremony of a home.

---

## Why the second consumer

Gall's Law governs the timing: *a complex system that works grows from a simpler system that worked.* One consumer inside the host means the code is simply well-organized internal structure — the host is still the honest owner of the boundary. Two consumers from *outside* the host mean the capability has become shared infrastructure; the tree should say so with a directory, witnesses, and imports like every other module.

**One consumer** — keep the aspect inside. Refactoring for elegance alone is breadth thinning depth.

**Two external consumers** — graduate: create `module/`, move the source, update imports, add or split witnesses, record the graduation in a session log.

**Zero external consumers** — the aspect may still be first-class in the lexicon and in design prose; it simply has not earned a directory yet.

---

## What counts as a consumer

A **consumer** is a module (or a Pond application above the modules) that imports and relies on the aspect's public surface at build time or runtime — not the host using its own submodule internally.

Examples that **do not** count as a second consumer:

- Mantra calling `weave.rye` inside `mantra/src/` — the host using its own aspect.
- Tests inside the host module that exercise the aspect — still one module's discipline.

Examples that **do** count:

- Comlink serving named data by reading Weave's fact shape directly — a second module depending on Weave's API.
- A second Pond app importing Skate grid types without going through Brushstroke — would trigger Skate graduation (hypothetical; has not happened).

When in doubt, ask: *If the host module vanished, would this other module still need this capability under its own name?* Two yes answers from distinct hosts mean graduation is due.

---

## Weave — the recorded trigger

Weave is first-class in meaning — the append-only DAG of immutable signed facts beneath Mantra and Comlink. Today it lives as an **aspect of Mantra** at `mantra/src/weave.rye`.

**Graduation day (recorded, not yet arrived):** when Comlink serves named data straight from Weave, Weave earns `weave/` as its own home, and Mantra imports it like everyone else. Until that second consumer runs, moving Weave early would split Mantra's first lap without a boundary the tree has proven.

---

## Dexter — the recorded trigger

**Dexter** names keyboard-from-window — the capability that composes key events into lines and feeds the Rishi session the drawn terminal already mirrors. Kaeden nodded placement **`pond/apps/`** on `20260703.131812`: the keyboard seam extends Lap 3's poll loop in [`pond/apps/drawn_terminal.rye`](../../pond/apps/drawn_terminal.rye), not a freestanding `dexter/` module with one consumer.

**Graduation day (recorded, not yet arrived):** when a **second external consumer** imports keyboard-from-window — a Pond application beyond the drawn terminal, a TAME editor surface, or the sovereign framebuffer backend's input path — Dexter earns `dexter/` as its own module home, and the drawn terminal imports it like everyone else. Until that second consumer runs, seeding `dexter/` early would stand a one-consumer module against the rule this page states.

---

## Graduation checklist (mechanical)

When the second consumer lands:

1. Create `module/` with README, seed, and witnesses.
2. Move or extract the aspect source; preserve git history when practical.
3. Publish a minimal public surface — only what consumers need.
4. Update the host to import the graduated module.
5. Wire parity witnesses for the new home; keep host witnesses green.
6. Update `LEXICON.md` and the active-designing brief if the one-line meaning shifts.
7. Session-log the graduation with stamp and parity count.

---

## What the rule is not

- **Not a ban on aspects.** Aspects keep the tree honest while Gall's Law still favors small wholes.
- **Not instant rename.** Lexicon names may lead implementation; directories follow consumers.
- **Not symmetry with opening lines or TAME witnesses.** Graduation is architectural judgment, witnessed in prose and parity, not a textual lint.

---

*May every aspect know its host. May every module earn its door only when two hands outside knock.*
