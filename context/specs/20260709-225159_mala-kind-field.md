## 1. What Lives Today

| Binding | Where |
|---------|--------|
| `mala:mint` | Exact memo or `mala:mint …` prefix |
| `mala:send` | Same shape |
| `mala:receipt` | Same shape |
| Fold | `mala_core` dispatches on those helpers |

Receipt-core `FactFields` carry `from`, `to`, `amount`, `memo`, `stamp` — no `kind` column. WOV roots and exits use their own memo prefixes (`wov:root`, `wov:exit`) on the same fact grammar.

---

## 2. Two Candidates (Claude Weighs)

| Choice | Shape | Pros | Costs |
|--------|-------|------|-------|
| **K0 — Keep memos** | Prefixes remain the binding through M3 | Zero migration; goldens and Comlink payloads stay; matches "memo as the kind" already on metal | Parsers must keep prefix discipline; crowded memos if free text grows |
| **K1 — Dedicated `kind`** | New canonical field (e.g. `kind mala.mint`) beside or instead of prefix | Clearer for many fact types; easier indexes; less stringly typing | Touches `receipt_core`, every MALA/WOV/OA golden, wire guests, parity; needs a migration story |

**Bench leaning:** **K0 through M3.** A dedicated `kind` earns its lap when a **second consumer** needs typed dispatch that memos cannot carry cleanly (graduation rule) — or when free-text memos and typed kinds must coexist without prefix collisions. Digital land (M3) can still open on memo prefixes if plot facts use a new prefix (`mala:plot` or similar) rather than a schema bump.

---

## 3. When K1 Would Open

Open a lap-open only if one of these is true:

1. A module outside MALA must filter facts by type without parsing memo text.
2. Memo bodies need arbitrary human text **and** a machine kind at once.
3. Claude/Kaeden affirm a one-time schema bump with a dated migration witness.

Until then, new MALA fact types add a **named memo prefix** and a `memo_is_*` helper — same pattern as mint/send/receipt.

---

## 4. Explicitly Out

- Changing WOV or Open Asks memos in the same lap as K1
- Removing memo prefixes while dual-writing `kind` (pick one binding per era)
- Claiming K0 is forever — it is the default until a second consumer appears
