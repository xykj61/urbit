## Parity gate — what changes (read before executing)

Today `tools/parity.rish` runs each witness twice:

- **Baseline arm:** `RYE_LIB=vendor/zig-toolchain/lib`
- **Strengthened arm:** default `rye/lib` (strengthened `std`)

After re-fork, **both arms compile against the same pristine `std` bytes**. Parity becomes **trivially GREEN** — it no longer witnesses divergence.

**That is not failure.** The differential parity gate's job was to prove strengthened `std` matched baseline behavior. Once `std` is pristine, that question is answered permanently: **there is no strengthened fork to witness.**

### What to do with parity

| Artifact | After re-fork |
|----------|----------------|
| `tools/parity.rish` | **Retire or repurpose** — e.g. rename to `witness-suite.rish` running witnesses once against pristine `std`; update message to drop "strengthened vs baseline" framing |
| `tools/parity-selftest.rish` | **Repurpose** — prove a test harness catches divergence if someone re-introduces a forked file (shadow-copy tamper on a symlinked tree) |
| `rye/tests/*.rye` (116 witnesses) | **Keep** — behavior tests against `std`; still valuable as regression suite |
| `additive-gate.rish` | **Keep** — still useful if any local Zig patches return; likely quiet on pristine |

**Do not mistake post-re-fork GREEN parity for proof of anything** — document this in `rye/README.md` and `rye-learning-process/ALMANAC.md` when executing.
